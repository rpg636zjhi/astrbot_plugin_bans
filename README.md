# AstrBot 黑名单插件 (astrbot_plugin_Bans)

这是一个用于 AstrBot 的插件，旨在帮助机器人管理员便捷地管理黑名单用户，并自动清理包含黑名单成员的群聊和好友。插件基于 NapCat (aiocqhttp) 的 API 实现，支持手动添加/移除黑名单、检查所有群聊和好友、定时自动巡查等核心功能。

---

## 功能特性

- **黑名单管理**  
  通过指令添加或移除 QQ 号到黑名单，数据持久化存储。

- **自动踢群 & 删除好友**  
  加入黑名单后，机器人会自动删除与该用户的好友关系；执行全局检查时，会自动退出所有包含黑名单成员的群聊，并在群内发送通知。

- **手动全量检查**  
  一键扫描所有群聊和好友列表，处理所有黑名单用户，并生成详细报告。

- **定时自动巡查**  
  可设置间隔小时数，让机器人定时执行全量检查，确保黑名单用户被及时清理。

- **操作结果通知**  
  操作结果会通过私聊发送给命令发起者，避免因退群导致消息丢失。

---

## 命令列表

所有命令均需在 AstrBot 可接收消息的会话（群聊或私聊）中发送，机器人会以被动回复或私聊方式返回结果。

| 命令格式 | 说明 |
|---------|------|
| `/bans <QQ号>` | 将指定 QQ 号加入黑名单，并立即删除与该用户的好友关系。 |
| `/debans <QQ号>` | 从黑名单中移除指定 QQ 号（不会自动加回好友）。 |
| `/banslist` | 列出当前所有黑名单 QQ 号及总数。 |
| `/CAllbans` | **C**heck **All** – 手动触发全量检查：<br> - 扫描所有群聊，收集含有黑名单成员的群；<br> - 发送汇总报告（当前会话）；<br> - 向目标群发送退群通知并退出；<br> - 删除所有黑名单好友；<br> - 最终结果通过私聊发送给命令发起者。 |
| `/Tbans <小时数>` | 启动定时巡查任务，每隔指定小时数自动执行一次全量检查（与 `/CAllbans` 逻辑相同）。再次执行该命令会重置定时器。 |

## 数据存储

插件数据存放在 AstrBot 数据目录下的 `plugin_data/astrbot_plugin_Bans/ban_list.json` 文件中，格式为 JSON 列表，例如：

```json
["123456789", "987654321"]
```

首次运行时会自动创建该文件。

## 依赖与兼容性

- **AstrBot 版本**：需支持插件系统及 `AstrMessageEvent`。
- **平台依赖**：本插件针对 **NapCat (aiocqhttp)** 平台设计，使用 `AiocqhttpMessageEvent` 调用 OneBot v11 API。若在其他平台使用，相关 API 调用将失败。
- **Python 依赖**：仅需标准库（`asyncio`, `json`, `pathlib`）及 AstrBot 自带模块。

## 安装方法

1. 将本插件文件夹复制到 AstrBot 的 `plugins` 目录下。
2. 重启 AstrBot
3. 确保机器人已连接至 NapCat 且具有相应的权限

---

## 注意事项

- **权限要求**：机器人必须拥有管理员权限或足够权限才能获取群成员列表、退群及删除好友。
- **消息发送**：退群通知和汇总报告依赖于消息发送 API，若群内无法发送（如被禁言），则可能失败，但不会影响退群操作。
- **私聊通道**：最终结果优先通过私聊发送给命令发起者，请确保机器人已添加发起者为好友，否则私聊可能失败（此时会尝试发送至原会话，若已退群则可能丢失）。
- **定时任务**：定时任务在插件生命周期内持续运行，卸载插件或重启 AstrBot 时会自动取消。
- **并发安全**：定时任务与手动命令共享同一个黑名单数据，操作时注意避免冲突（插件内部未加锁，但通常单个用户操作频率低，不会产生严重问题）。
